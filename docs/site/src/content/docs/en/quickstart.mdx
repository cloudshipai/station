# Quick Start

Get Station up and running in minutes and deploy your first intelligent sub-agent.

## Installation

### Automatic Install (Recommended)

```bash
curl -fsSL https://raw.githubusercontent.com/cloudshipai/station/main/install.sh | bash
```

This script automatically detects your platform and installs the latest version.

### Manual Installation

```bash
git clone https://github.com/cloudshipai/station
cd station
make local-install
```

This will build Station and install it to `~/.local/bin/stn`.

## Setup Workflow

### 1. Initialize Station

```bash
stn init
```

This creates the Station configuration directory and initializes the database.

### 2. Load MCP Tools

Station works with Model Context Protocol (MCP) servers. Load some essential tools:

```bash
# Create filesystem MCP config
cat > filesystem-mcp.json << 'EOF'
{
  "name": "Filesystem MCP",
  "description": "Basic filesystem operations",
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem@latest",
        "{{ .ALLOWED_PATH }}"
      ]
    }
  }
}
EOF

# Load the configuration
stn load filesystem-mcp.json --env default

# Create variables for template resolution
echo "ALLOWED_PATH: /home/user/projects" > ~/.config/station/environments/default/variables.yml

# Sync to discover tools
stn sync default
```

### 3. Create Your First Agent

```bash
stn agent create \
  --name "System Monitor" \
  --description "Monitor system resources and alert on issues"
```

### 4. Test Your Agent

```bash
stn agent run 1 "Check current disk usage and memory status"
```

### 5. Connect to Claude Desktop

Add Station to your Claude Desktop MCP configuration:

**Location:**
- **macOS**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Windows**: `%APPDATA%\\Claude\\claude_desktop_config.json`

**Configuration:**
```json
{
  "mcpServers": {
    "station": {
      "command": "stn",
      "args": ["stdio"]
    }
  }
}
```

Restart Claude Desktop and you can now create and run agents through natural conversation.

## Access Methods

Station provides multiple ways to interact with your agents:

### Command Line

```bash
# Direct agent execution
stn agent run 1 "task description"

# List agents
stn agent list

# Check system status
stn status
```

### SSH/TUI Access

```bash
# Access admin interface via SSH
ssh admin@localhost -p 2222
```

Interactive terminal interface for managing agents and viewing logs.

### REST API

```bash
# Queue agent execution
curl -X POST http://localhost:8080/api/v1/agents/1/queue \
  -H "Content-Type: application/json" \
  -d '{"task": "Check system status"}'
```

### WebSocket

Real-time updates and streaming responses for agent execution and system events.

## Development Workflow

1. **Create Agent** - `stn agent create` to define your agent
2. **Test Locally** - `stn agent run` to test functionality
3. **Export Template** - `stn agent export` to create reusable templates
4. **Version Control** - `git add & commit` for your agent configurations
5. **Deploy** - `stn template install` in production environment
6. **Monitor** - Use SSH/API access to monitor execution

## System Requirements

- **OS**: Linux, macOS, Windows
- **Memory**: 512MB RAM minimum, 1GB recommended  
- **Storage**: 200MB for runtime, 1GB+ for production data
- **Database**: SQLite (built-in, no setup required)
- **Network**: Outbound HTTPS for AI providers and tool integrations
